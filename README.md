# Onboarding Agent

A built-from-scratch onboarding agent that learns from a company's information what onboarding requires for a specific hire, then runs the lifecycle from contract draft through 30-day check-in.

Built as a teaching artifact: every choice in this repo is meant to be readable and copyable. If you walked away from the workshop unsure where to start, start here.

**Theme:** an agent enforces structure, the human keeps judgment.

---

## What's in here

- `CLAUDE.md` / `AGENTS.md` — the agent's identity and operating rules. Read by the runtime on every turn.
- `.claude/commands/` — five slash commands (skills). One per phase of the onboarding lifecycle.
- `companies/lattice-loom/` — a fully fictional Series A B2B SaaS. The agent reads this to ground its outputs in a real company shape.
- `companies/lattice-loom/hires/marcus-chen/` — a fictional new hire. Intake is pre-built; everything else is generated by the skills.
- `templates/` — boilerplate offer letter, welcome email, manager prep, 30-day check-in.
- `reference/onboarding-principles.md` — the doctrine. The agent reads this before every skill.
- `docs/agent-primer.md` — the workshop curriculum, written up.
- `docs/why-this-shape.md` — teaching artifact. Why every file in this repo exists.

Lattice Loom and everyone in it is fictional. Any resemblance to a real company is coincidental.

## Run it on the platform you have

This agent is a folder of markdown files. The platform-specific glue is small. Pick yours:

### Claude Code

1. `git clone` this repo
2. Open the directory in Claude Code
3. Slash commands auto-load from `.claude/commands/`
4. Run `/learn-company lattice-loom marcus-chen` to start

### OpenAI Codex (or any agent that reads `AGENTS.md`)

1. `git clone` this repo
2. Open the directory in Codex
3. Codex reads `AGENTS.md` at root automatically
4. Invoke a skill by name — e.g., "run learn-company on lattice-loom marcus-chen" — Codex will follow the `.claude/commands/learn-company.md` procedure

### Google Antigravity

1. `git clone` this repo
2. Open the directory; Antigravity reads `AGENTS.md` and the markdown files
3. Same invocation pattern as Codex

### Anywhere else (ChatGPT projects, Cursor, generic LLMs)

The skills are plain markdown procedures. You can paste any skill file's content into a chat and ask the model to follow it. The agent works without any tooling — it's just structured prose. Tooling makes it ergonomic, not possible.

## The five skills

Run them in order, or any one independently after `/learn-company`:

| Skill | What it does |
|---|---|
| `/learn-company [company] [hire]` | Reads the company files + hire intake, writes a tailored 30-60-90 plan |
| `/send-contract [hire]` | Drafts an offer letter; **stops for human y/n before mock-sending** |
| `/preboard [hire]` | Day -7 → Day 0 task list with owners (HR / IT / Manager / Agent) and draft artifacts |
| `/first-day [hire]` | Day 0 morning Slack post + Week 1 calendar + manager handoff brief |
| `/thirty-day [hire]` | 30-day check-in email + internal retention indicators |

Two human-judgment checkpoints are mandatory by design:

1. Before any contract mock-send (`/send-contract`)
2. Before any employee-facing draft goes out (Slack post, Week 1 calendar invites in `/preboard` and `/first-day`)

## Demo it in 6-14 minutes

See [the runbook](#the-7-beat-demo) below. The 6-min version cuts beats 4 and 5; the 14-min version is the full flow.

### The 7-beat demo

1. **Setup** (1 min) — show `profile.md`, `handbook.md`, `intake.md`. "Here's what the agent knows."
2. **`/learn-company lattice-loom marcus-chen`** (3 min) — agent generates `plan.md`. Narrate three things it specifically picked up (hybrid Boston, on-call by Day 60, HIPAA-naïve ramp).
3. **`/send-contract marcus-chen`** (2 min) — agent drafts, stops, asks y/n. "Agent enforces structure. I keep judgment."
4. **`/preboard marcus-chen`** (3 min) — agent generates owner-tagged task list. "Half of these aren't the agent. The agent coordinates."
5. **`/first-day marcus-chen`** (2 min) — agent generates Slack post + Week 1 calendar + manager brief. "The manager brief is the load-bearing one."
6. **`/thirty-day marcus-chen`** (2 min) — two-section file, one for the hire, one internal. "Patterns surface. Decisions stay with humans."
7. **Close** (1 min) — "An agent is a loop with judgment, tools, and boundaries."

A FigJam version of this flow lives [here](https://www.figma.com/board/Z4LKLR2oGhDpWVzbmceYKY).

```bash
cd companies/lattice-loom/hires/marcus-chen

```

The intake stays. Everything else regenerates.

## How to extend it

### Add a new hire to Lattice Loom

```bash
mkdir -p companies/lattice-loom/hires/[new-hire-slug]
cp companies/lattice-loom/hires/marcus-chen/intake.md \
   companies/lattice-loom/hires/[new-hire-slug]/intake.md
# Edit the intake. Then run the five skills against [new-hire-slug].
```

### Add a new company

```bash
mkdir -p companies/[new-company]/hires
# Author profile.md, handbook.md, onboarding-template.md
# following the lattice-loom shape.
```

The skills are company-agnostic. They derive the company from the hire's path.

### Add a new skill

Drop a markdown file into `.claude/commands/`. Use the existing skills as a shape guide: trigger description on line 1, then **Step 1 — Read**, **Step 2 — [Verb]**, **Step 3 — Write**, **Step 4 — [Mock-send / human checkpoint]**.
